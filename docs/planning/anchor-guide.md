HUMAN PLANNING DOC GUIDE

Purpose:
Create a short human-written doc before /preplan to anchor product intent.

Use when:

- Starting a new project
- Planning a large feature area
- Scope is bigger than a single chunk

This doc should:

- Define product purpose and user outcome
- Identify core flows
- Define v1 scope (in/out)
- Establish system boundaries
- Name domain concepts
- Define constraints and rules
- Reduce ambiguity before AI planning

This doc should NOT:

- Include file structure
- Include implementation details
- Solve every edge case
- Replace /plan or /build

Quality rule:
Keep it short, clear, and focused on intent.
Reduce guessing for /preplan.
